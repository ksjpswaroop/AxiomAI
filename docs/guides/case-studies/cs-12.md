# CS-12 User Guide — Data Governance

**Tier:** 2 · **ID:** `cs-12`

## Problem
Analysts request access to regulated data without proper role.

## Demo story
Data analyst denied access to PII/PCI Salesforce dataset.

## How to run (no code)

1. Open **Case Study Gallery** in the investor console
2. Select **Data Governance**
3. Choose a scenario:
   - **Access denied — wrong role** (`default`) → expected: `DENY`
4. Click **Run demo** — view outcome, proof, and data sources

## API

```bash
curl -X POST http://localhost:8000/case-studies/cs-12/run \
  -H "Content-Type: application/json" \
  -d '{"scenario": "default"}'
```

## Data sources

- [NIST SP 800-122 PII](https://csrc.nist.gov/publications/detail/sp/800-122/final) — Public domain / synthetic demo
- [PCI DSS access control](https://www.pcisecuritystandards.org/) — Public domain / synthetic demo
